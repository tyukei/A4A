# Contributing to A4A

A4A (Agent for Agent) プロジェクトへようこそ！
自分のオリジナルのエージェントを作成し、フォークしてプルリクエストを作成して共有していただくことを大歓迎しています。

## 開発の始め方

### 1. セットアップ
プロジェクトの最新ツールセットを使用するために `uv` ツールチェーンを推奨しています。

```bash
# リポジトリのクローン
git clone https://github.com/tyukei/A4A.git
cd A4A

# 依存関係の同期
uv sync
source .venv/bin/activate

# 環境変数の設定
cp agent_4_agent/.env.example agent_4_agent/.env
# agent_4_agent/.env に GEMINI_API_KEY を設定してください
```

### 2. 新しいエージェントの作成
新しいエージェントを作成する際は、以下の構成に従うことをお勧めします。

```text
agent_4_agent/
 ├── agents/
 │    └── {your_agent_name}/
 │         ├── agent.py      # エージェントの定義
 │         ├── tools/         # ツール定義
 │         └── subagents/     # サブエージェント定義
```

### 3. ローカルでのテスト
Google ADK の Web インターフェースを使用して、作成したエージェントをテストできます。

```bash
adk web
```
ブラウザで `http://127.0.0.1:8000` を開き、左上のメニューから自分のエージェントを選択して対話してください。

## 変更の提案（プルリクエスト）

このプロジェクトでは GitHub Actions による自動化を導入しています。

1. **プルリクエストの作成**:
   - 自分のブランチから `main` ブランチに対して PR を作成します。
   - GitHub の **Actions** タブにある **"Manual PR Creator"** ワークフローを実行すると、AI がコミットログから自動的に PR タイトルと説明文を生成してくれます。
2. **レビュー**:
   - メンテナーが内容を確認します。

## リリースの自動化

`main` ブランチにプルリクエストがマージされると、システムが以下の処理を自動で行います：
- `pyproject.toml` のパッチバージョンをインクリメント。
- Google Gemini がマージされた内容を分析し、**「新機能・変更点」**のサマリーを日本語で作成。
- 新しい Git タグと共に GitHub Release を自動的に作成。

## コーディング規約
- **言語**: Python 3.12 以上
- **スタイル**: 可読性を重視し、複雑なロジックにはコメントを添えてください。
- **日本語化**: ユーザーに直接触れるメッセージやリリースノートのサマリーは日本語を基本としています。
